# SvelteKit Development Standards

## Rules and Guidelines

### Code Editing
1. Ask clarifying questions if any part of the prompt or requirements is ambiguous.
2. Make code changes strictly according to the provided prompt and requirements.
3. Avoid making unrelated changes or optimizations unless explicitly requested.


## UI and Visual Design

1. Ensure the page is responsive and visually appealing.
2. Always follow the existing design system and styles found in `src/lib/components/`.
3. Consolidate and extract reusable components where applicable.


## API Integration

Refer to `src/routes/api` for existing API endpoints and best practices for integrating with backend services.

## Data Types and Models

Refer to `src/lib/types` for existing data models and schema definitions.
